# 1E：法宝 / 丹药 / 经济系统设计

> 基于《凡人修仙传》剧本提取，覆盖法宝体系、丹药体系、修仙经济/灵石体系三大模块。
> 本文档是 ITEMS_AND_ECONOMY.md 的详细扩展版，包含完整数值设计与 LPC 实现指引。

## 一、法宝体系

### 1.1 法宝等级体系（从低到高）

| 等级 | 说明 | 适用境界 | 典型例子 |
|------|------|---------|---------|
| **法器** | 最低阶修士武器，分下品/中品/上品/极品 | 炼气期~筑基期 | 基础飞剑、低阶护甲 |
| **法宝** | 结丹以上修士本命物，分下品/中品/上品/极品 | 结丹期~元婴期 | 青竹蜂云剑（初阶） |
| **古宝** | 上古修士遗留，威能远超同阶法宝 | 结丹期~化神期 | 虚天鼎、天遁舟 |
| **通天灵宝** | 灵界最广泛的高阶宝物体系，蕴含法则之力 | 化神期~合体期 | 八门金光镜、万灵血玺 |
| **玄天之宝** | 蕴含完整天地法则，可逆转一界法则之力 | 大乘期~真仙 | 玄天斩灵剑、掌天瓶 |

**原著统计：**
- 「法器」提及 1626 次
- 「法宝」提及 1057 次
- 「通天灵宝」提及 527 次
- 「古宝」提及 315 次
- 「玄天之宝」提及 204 次

#### 法宝品质维度

- **材质**：铁精→银精→金精→庚精→玄铁→万年玄玉→炎金之精→异魔金→星辰钢→万载玄冰→天外陨石
- **禁制层数**：从数层到数十层不等，禁制越多威能越大
- **通灵程度**：普通→通灵→伪灵宝→真灵宝
- **剑灵**：高阶飞剑可孕育剑灵，大幅提升威能

### 2.3 炼丹体系

```
灵药采集 → 年份鉴别 → 药性搭配 → 炼丹（丹炉/火候/丹方） → 成丹（品质）
                     ↓                                    ↓
                药性冲突（失败）                    上/中/下三品
```

**炼丹三大要素：**

| 要素 | 说明 |
|------|------|
| **丹炉** | 低阶丹炉→高阶丹炉→虚天鼎（可炼丹）→玄天丹炉 |
| **火候** | 地火→丹火→婴火→天地灵火 |
| **丹方** | 药方传承（门派/拍卖会/遗迹获得） |
| **灵药年份** | 百年药→千年药→万年药→十万年年药（掌天瓶可催熟） |

**炼丹成功率影响因素：**
- 炼丹师等级（炼药术技能）
- 丹炉品质
- 火候控制
- 灵药年份与匹配度
- 数量与品质取舍（高年份灵药提升成功率）

### 2.4 掌天瓶与丹药经济

掌天瓶（九彩造化瓶）是游戏中**最重要的神器级物品**：

- **绿色灵液**：滴在灵药根部，一夜之间可增加百年药龄
- **银色光点**（后期）：时间法则相关，可加速时间
- **游戏设计意义**：
  - 掌天瓶持有者可大量产出高年份灵药→打破丹药稀有度平衡
  - 自然形成「炼丹专精」职业路线
  - 成为经济系统的核心变量（高年份灵药供应者）

## 三、经济/灵石体系

### 3.1 货币体系

#### 灵石等级与兑换

| 等级 | 兑换比例（低→高） | 主要使用者 | 获取难度 |
|------|-----------------|-----------|---------|
| **下品灵石** | 基础单位 | 炼气期~筑基期 | 日常产出 |
| **中品灵石** | 1 : 100（下品） | 结丹期~元婴期 | 稀有矿脉 |
| **上品灵石** | 1 : 100（中品） | 元婴期~化神期 | 高阶矿脉 |
| **极品灵石** | 可遇不可求 | 化神期以上 | 一个矿脉可能才出一块 |

**原著中提及频率：**
- 「灵石」提及 1331 次
- 「极品灵石」提及 83 次
- 「上品灵石」提及 3 次
- 「中品灵石」提及 2 次

#### 极品灵石的特殊性

**凡间货币：** 金银铜钱与灵石不直接挂钩，玩家早期在凡人江湖使用。

- 蕴含巨量精纯灵气，对冲破瓶颈有**不可思议的神效**
- 一颗极品灵石木属性→可辅助化神突破
- 跨大陆传送阵消耗极品灵石数量惊人（数百颗级别）
- 在灵界虽然比人界常见，但仍属珍宝
- 高阶修士的硬通货，拍卖会大额交易专用结算单位
- 可用于驱动超级传送阵、高阶傀儡、大型法阵

### 3.4 经济循环

```
输入：打怪掉落/采集挖矿/任务奖励
       ↓
玩家获得原材料（灵药、矿石、妖兽材料）
       ↓
加工：炼丹/炼器/制符/傀儡制造
       ↓
成品：丹药/法宝/符箓/傀儡
       ↓
自用或交易（坊市/拍卖行/店铺）
       ↓
灵石产出/消耗循环
```

**灵石消耗途径：**

| 途径 | 说明 | 涉及阶段 |
|------|------|---------|
| 修炼消耗 | 聚灵阵抽灵、闭关消耗 | 全境界 |
| 突破辅助 | 极品灵石辅助突破瓶颈 | 化神及以上 |
| 购买资源 | 丹药、法宝、材料 | 全境界 |
| 传送费用 | 跨区域/跨大陆传送阵（需极品灵石） | 元婴以上 |
| 傀儡驱动 | 高阶傀儡运转需灵石供能 | 结丹以上 |
| 法阵消耗 | 防御阵/攻击阵/传送阵灵石消耗 | 全境界 |
| 宗门贡献 | 门派运营、任务奖励 | 全境界 |

**灵石产出途径：**

| 途径 | 稳定性 | 风险 |
|------|--------|------|
| 灵石矿脉开采 | 稳定但竞争激烈 | 其他势力抢夺 |
| 任务/悬赏奖励 | 稳定 | 低 |
| 妖兽/怪物掉落 | 不稳定 | 战斗风险 |
| 炼丹/炼器制售 | 高回报 | 成功率/市场波动 |
| 倒卖物品（低买高卖） | 看市场 | 资金占用 |
| 掌天瓶催熟灵药** | 极高回报 | 神器保密风险 |

| 风险类型 | 触发条件 | 后果 |

| **极品灵石争夺** | 极品灵石消息走漏 | 高等级修士追杀 |

### 3.6 关键经济事件（原著参考）

1. **碧灵岛极品灵石争夺** — 海族与人族为一块极品木属性灵石爆发大规模冲突，死伤众多
2. **灵界拍卖会** — 通天灵宝竞拍，价码数百万~数千万灵石
3. **跨大陆超级传送阵** — 需要天量极品灵石驱动，少数大势力才能承担
4. **掌天瓶催熟灵药** — 韩立借此获取大量高年份灵药，成为炼丹师资源基础

### 3.7 符箓体系

| 等级 | 示例 | 使用者 |
|------|------|--------|
| 初级下阶符 | 冰弹符、火弹符 | 炼气期 |
| 初级中阶符 | 连珠雷符 | 炼气后期~筑基 |

符宝是原著中跨境界战斗的关键道具——低阶修士持有高阶符宝可对战高境界敌人。

### 3.8 材料体系补充

**灵药/药材分类（按年份）：**
- 百年灵药 → 炼气~筑基期使用
- 千年灵药 → 结丹~元婴期使用
- 万年灵药 → 化神以上使用

**妖兽材料：**
击杀妖兽后获得的皮毛、血液、内丹、骨骼等。妖兽等级越高材料价值越高。妖丹可直接用于炼丹或辅助修炼。

**材料体系与经济的关联：**
```
采集/狩猎 → 原材料 → 加工（炼器/炼丹/制符）→ 成品 → 自用或售卖
                                                                  ↓
                                                             灵石流通
```

### 4.1 数据结构

```
宝物: inherit F_DBASE
  set("name")           — 名称
  set("treasure_type")  — 法器/法宝/古宝/通天灵宝/玄天之宝
  set("level")          — 下品/中品/上品/极品
  set("element")        — 金/木/水/火/土/风/雷/空间/时间
  set("forbidden_count") — 禁制层数
  set("attack")         — 攻击力加成
  set("defense")        — 防御力加成
  set("special")        — 特殊效果描述
  set("require_level")  — 使用所需境界

丹药: inherit F_DBASE
  set("name")
  set("pill_type")      — 炼气散/筑基丹/结金丹等
  set("stage")          — 对应修炼阶段
  set("effect")         — 修为增加量/突破成功率加成
  set("quality")        — 下品/中品/上品
  set("side_effect")    — 副作用（如丹毒累积）
  set("refine_level")   — 所需炼丹术等级

灵石: inherit F_DBASE
  set("name")
  set("stone_type")     — 下品/中品/上品/极品
  set("element")        — 无属性/金/木/水/火/土
  set("value")          — 兑换价值（以下品为1单位）
```

### 4.2 关键守护进程建议

