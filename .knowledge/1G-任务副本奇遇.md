## 一、任务体系总览

### 1.1 任务分类

| 类型 | 说明 | 可重复 | 典型例子 |
|------|------|--------|---------|
| **主线任务** | 跟随韩立修炼旅程推进，不可跳过，解锁新区域/新境界 | 否 | 入七玄门→拜黄枫谷→血色禁地→虚天殿 |
| **支线任务** | 与主线并行的可取舍任务，影响声望/奖励 | 部分可重做 | 墨府恩怨、太南谷坊市交易、妙音门叛变 |
| **日常/周常任务** | 定期刷新的重复任务，获取基础资源 | 是 | 采集灵药、狩猎妖兽、门派杂务 |
| **势力任务** | 特定势力声望解锁的专属任务链 | 可重复但有限制 | 星宫任务、魔道任务、灵界种族任务 |
| **奇遇任务** | 低概率触发的一次性事件 | 否（唯一） | 掌天瓶认主、万年灵乳发现 |
| **副本任务** | 组队/单人挑战秘境的任务化入口 | 按刷新周期 | 血色禁地试炼、虚天殿夺宝 |

### 1.3 任务链机制

- **串行链**：前置任务完成后自动接续下一个（主线严格串行）
- **分支链**：在同一节点可选择不同方向（如越国篇可选择先做门派任务还是先探索坊市）
- **并行链**：主线+支线可同时推进，互不影响
- **条件链**：满足声望/境界/物品条件才解锁后续
- **限时链**：部分任务有游戏内时间限制（如血色禁地必须在限定游戏日内完成）

## 三、副本/秘境体系

### 3.1 副本分类

| 类型 | 说明 | 可重复性 | 典型副本 |
|------|------|---------|---------|
| **剧情副本** | 主线必经、剧情驱动的单次秘境 | 一次性 | 血色禁地（主线）、七玄门密室 |
| **挑战副本** | 高难度、多关卡、周期重置 | 按周期 | 虚天殿、地渊各层、广寒界 |
| **探索副本** | 开放式地图，探索收集 | 可重复但有限制 | 魔金山脉、外星海岛屿 |
| **试炼副本** | 宗门/势力内部考验 | 定期开放 | 升仙大会、门派晋升试炼 |
| **事件副本** | 服务器事件触发的限时副本 | 事件驱动 | 魔界入侵、碧灵岛大战 |

### 3.2 人界副本详解

#### 3.2.1 血色禁地

| 属性 | 详情 |
|------|------|
| **推荐境界** | 筑基中期—筑基后期 |
| **准入条件** | 持有七派联合发放的禁地令牌（完成升仙大会后获得） |
| **人数限制** | 单人/组队（最多6人） |
| **刷新周期** | 5年（游戏内时间），每次开启持续30天 |
| **副本结构** | 三层环形结构：外围森林→中部遗迹→核心地下洞窟 |
| **关键区域** | ①外围：迷瘴森林（随机毒雾）<br/>②中层：遗迹广场（遭遇其他队伍PVP区域）<br/>③核心：地下洞窟（BOSS墨蛟巢穴） |
| **产出列表** | 百年灵药（紫猴花、天灵果、蚀骨菇等）、墨蛟材料（可制符宝）、妖兽内丹 |

**怪物配置：**
- 外层：1-2级妖兽（毒蛇、巨蚊、毒蝎群）
- 中层：2-3级妖兽（石傀儡、吸血蝠群）
- 核心：BOSS墨蛟（3级妖兽）+ 两只护卫妖兽

**设计要点：**
- 禁地内天地灵气变异，法力恢复速度减半
- 各派弟子可互相攻击（PVP区域），击杀后掉落对方采集物品
- 禁地关闭前未退出者会被永久困入（传送至禁地中心，等待下次开启——可选设计为死亡）
- 首次通关成就：血色禁地幸存者

#### 3.2.2 虚天殿

| 属性 | 详情 |
|------|------|
| **推荐境界** | 结丹期—元婴期 |
| **准入条件** | 持有虚天残图（通过探索乱星海获得碎片合成，或击杀持有者掉落） |
| **人数限制** | 单人/组队（自由组合） |
| **刷新周期** | 300年（游戏内时间）/ 或根据服务器节奏调整为定期开启事件 |
| **副本结构** | 五大关卡层层递进 |
| **产出列表** | 补天丹（最高价值）、古宝、功法残篇、寿元果、九曲灵参、万年灵乳 |

**五大关卡详解：**

| 关卡 | 名称 | 难度 | 内容 |
|------|------|------|------|
| 第一关 | 鬼冤之地 | 结丹初期可过 | 鬼雾区域，低能见度，厉魄无数（需克制鬼物法器/符箓） |
| 第二关 | 冰火道 | 结丹中期 | 极寒极热交替通道，冰火两重天禁制，触发即受伤；通道尽头是灵药区 |
| 第三关 | 传送迷宫 | 结丹后期 | 密室选择（古宝/丹药/功法三选一），每人限取一件，取后传送至幻境 |
| 第四关 | 极妙幻境 | 结丹后期 | 心魔幻象考验，失败则迷失（可救援），通过后进入内殿 |
| 内殿 | 虚天殿核心 | 元婴期 | 五层结构，每层均有禁制/BOSS；最终BOSS虚天鼎守护禁制，获补天丹和虚天鼎 |

**虚天残图获取方式：**
- 乱星海探索随机事件（30%概率）
- 击杀特定NPC掉落（星宫执法长老、逆星盟头目）
- 天星城黑市购买（天价）
- 集齐四块碎片合成一张完整残图

**设计要点：**
- 虚天殿是游戏核心玩法，建议设计为服务器定期事件（每现实1-2周开启一次）
- 内殿仅元婴期可安全进入，结丹期玩家进内殿死亡率极高
- 传送迷宫关卡是资源获取的关键节点，三选一不可后悔

#### 3.2.3 昆吾山

| 属性 | 详情 |
|------|------|
| **推荐境界** | 元婴后期—化神期 |
| **准入条件** | 完成越国篇主线，通过大晋正道盟/魔道六宗推荐 |
| **刷新周期** | 不定期（魔界裂缝活跃时触发） |
| **内容** | 镇压魔界裂缝，击杀封印魔族 |

**设计要点：**
- 昆吾山是正魔双方共同关注的封印地
- 副本内含魔界裂缝，定期刷新魔族怪物
- 产出魔核（元婴期法宝炼制材料）
- 通关后可获得化神突破线索

### 3.4 副本通用规则

```lpc
// 副本实例的生命周期
mapping instance = ([
  "id":           "dungeon_xue_se_001",
  "template":     "xue_se_jin_di",           // 副本模板ID
  "status":       DUNGEON_ACTIVE,             // 状态：准备/进行中/结算/关闭
  "creator":      player_name,               // 创建者（队长）
  "members":      ({ "playerA", "playerB" }), // 当前成员
  "created_time": time(),                     // 创建时间戳
  "expire_time":  time() + 2592000,           // 到期时间(30天游戏内)
  "state":        ([
    "current_floor":  1,
    "boss_alive":     true,
    "cleared_rooms":  ({}),
    "loot_taken":     ({}),
  ]),
]);
```

**副本通用约束：**
1. **准入验证**：境界、道具、任务进度、声望四级校验
2. **队长权限**：创建副本、邀请成员、踢出成员、解散
3. **退出机制**：正常传送退出（在安全区使用传送符）/ 死亡退出 / 强制踢出
4. **产出分配**：跟随（各自拾取）/ 队长分配 / 需求优先
5. **死亡惩罚**：副本内死亡不掉落装备（避免玩家损失过大），但失去继续探索资格
6. **进度保存**：退出后副本进度保留N小时（可重新加入继续）

### 5.3 声望增减规则

**增加声望的途径：**
- 完成该势力发布的任务（主要来源，每次+10~+100）
- 上缴该势力需要的珍稀物品（单次+20~+200）
- 在势力战争中为该势力出力（单次+50~+500）
- 解救该势力的重要NPC（单次+200~+500）

**减少声望的途径：**
- 击杀该势力成员（主动攻击，每次-50~-200）
- 拒绝/失败该势力任务（单次-20~-100）
- 加入敌对阵营（批量调整）
- 盗窃该势力财产（被发现后每次-30）

**声望惩罚机制：**
- 声望≤-500：该势力对玩家宣战，派遣追杀者
- 声望≤-100：无法进入该势力控制区域
- -100 < 声望 < 0：可进入但不受欢迎，交易价格上浮50%

## 七、LPC 实现指引

### 7.1 任务守护进程（Quest Daemon）

参考现有[[quest-system]]架构：

```
/quest/
├── quest/              # 任务守护进程目录
│   ├── quest_d.c       # 主任务守护进程(全局任务列表)
│   ├── main_quest.c    # 主线任务注册器
│   └── side_quest.c    # 支线任务注册器
├── instance/           # 副本实例管理
│   ├── dungeon_d.c     # 副本守护进程
│   ├── dungeon_util.c  # 副本工具函数
│   └── template/       # 副本模板
│       ├── xue_se_jin_di.c
│       ├── xu_tian_dian.c
│       └── di_yuan.c
└── encounter/          # 奇遇事件引擎
    ├── encounter_d.c   # 奇遇守护进程
    └── events/         # 具体奇遇事件定义
```

### 7.2 关键守护进程职责

**quest_d.c（任务守护进程）：**
- 全局任务注册表（所有任务的mapping索引）
- 玩家任务进度查询/更新接口
- 任务状态管理（未接/进行中/已完成/失败）
- 定时任务检查（日常刷新/限时任务倒计时）

**dungeon_d.c（副本守护进程）：**
- 副本模板注册
- 副本实例创建/销毁
- 准入条件检查代理
- 副本进度管理

**encounter_d.c（奇遇守护进程）：**
- 奇遇事件注册
- 概率轮询（定时检测玩家状态，按概率触发奇遇）
- 奇遇状态管理（选择分支/奖励发放）
